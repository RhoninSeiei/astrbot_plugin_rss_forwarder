# 路线图

## 当前重点

1. 继续压缩重复推送的边界情况，尤其是跨源同文、发布时间缺失、图文组合变化等场景。
2. 继续完善日报功能，包括更多摘要样式与更清晰的运行状态展示。
3. 提升运维可观测性，补足 feed 健康状态、target 发送状态、近期开销与错误统计。
4. 维持 `docs/llm/` 的持续更新，使项目结构、线上环境与历史决策保持可追溯。

## 中期事项

1. 评估统一插件注册名与发布名，减少 `astrbot_rss` 与 `astrbot_plugin_rss_forwarder` 并存带来的维护复杂度。
2. 评估真正的 cron 调度支持，避免当前 `cron` 配置回退为 interval 轮询。
3. 评估更细的分发策略，例如按来源、关键字、时间段与图片条件做选择性发送。
4. 评估更完整的日报图片模板与分组展示方式。

## 长期事项

1. 扩展智能增强能力，包括网页扩展读取、图片增强提取与更稳定的内容总结。
2. 建立更完整的运行历史查询接口，便于面板查看最近发送、跳过与错误信息。
3. 视 AstrBot 主程序能力变化，补充更强的表单校验与更清晰的配置提示。
